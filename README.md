# AI智能体开发教学项目

## 项目概述

这是一个基于Python的AI智能体开发教学项目，旨在帮助学习者了解如何使用Python标准库与本地LLM（大型语言模型）进行交互。

## 项目结构

```
├── practice01/           # 练习1：基础LLM调用
│   └── llm_client.py     # 基础LLM客户端代码
├── practice02/           # 练习2：终端聊天界面
│   ├── chat_client.py    # 终端聊天客户端
│   └── chat_history.json # 聊天历史记录（自动生成）
├── practice03/           # 练习3：聊天历史压缩
│   ├── chat_client.py    # 带聊天历史压缩功能的客户端
│   └── chat_history.json # 聊天历史记录（自动生成）
├── practice04/           # 练习4：AnythingLLM集成
│   ├── chat_client.py    # 带AnythingLLM查询功能的客户端
│   └── chat_history.json # 聊天历史记录（自动生成）
├── venv/                 # 虚拟环境
├── .env                  # 环境配置文件
├── env.example           # 环境配置模板
├── .gitignore            # Git忽略文件
└── README.md             # 项目说明文档
```

## Python代码功能说明

### practice01/llm_client.py

**功能用途：**
- 调用本地LLM（LMStudio）服务器
- 发送聊天完成请求
- 接收和处理LLM响应
- 统计性能指标（token消耗、时间、速度）
- 友好的中文输出界面

**核心功能模块：**

1. **load_env()**
   - 从项目根目录读取.env配置文件
   - 解析环境变量
   - 处理文件不存在的错误

2. **count_tokens()**
   - 简单的token估算（基于空格分割）
   - 用于统计输入和输出的token数量

3. **call_llm()**
   - 主函数，协调整个LLM调用流程
   - 解析配置参数
   - 构建HTTP请求
   - 发送请求并接收响应
   - 计算性能指标
   - 格式化输出结果

**技术实现：**
- 使用Python标准库 `http.client` 进行HTTP/HTTPS请求
- 使用 `json` 模块处理JSON数据
- 使用 `time` 模块进行时间统计
- 使用 `os` 模块处理文件路径
- 使用 `urllib.parse` 解析URL

### practice02/chat_client.py

**功能用途：**
- 基于练习1的LLM调用基础
- 实现交互式终端聊天界面
- 支持流式输出响应
- 实现**工具调用功能**，包括：
  - 文件操作工具（列出目录、创建文件、读取文件等）
  - 网络访问工具（curl获取网页内容）

**核心功能模块：**

1. **文件操作工具函数**
   - `list_directory(directory_path)` - 列出目录下所有文件及属性
   - `rename_file(directory_path, old_name, new_name)` - 重命名文件
   - `delete_file(directory_path, file_name)` - 删除文件
   - `create_file(directory_path, file_name, content)` - 创建并写入文件
   - `read_file(directory_path, file_name)` - 读取文件内容

2. **网络访问工具函数**
   - `curl_fetch(url, method, headers, data, timeout)` - 通过curl访问网页并返回内容

3. **工具调用系统**
   - `SYSTEM_PROMPT` - 系统提示词，定义可用工具
   - `parse_tool_calls()` - 解析LLM响应中的工具调用指令
   - `execute_tool_call()` - 执行工具调用
   - 支持多轮工具调用直到任务完成

**技术实现：**
- 使用Python标准库 `subprocess` 调用系统curl命令
- 使用标记 `[TOOL_CALL]` 和 `[/TOOL_CALL]` 包裹工具调用指令
- 流式输出响应，提升用户体验
- 聊天历史自动保存到JSON文件

### practice03/chat_client.py

**功能用途：**
- 基于练习2的终端聊天界面
- 实现聊天历史自动压缩功能
- 当聊天记录超过5轮或长度超过3k时，主动触发LLM执行聊天记录总结
- 对前70%的内容进行压缩，保留最后30%的原文

**核心功能模块：**

1. **聊天历史管理**
   - `get_chat_history_length()` - 计算聊天历史的总长度
   - `summarize_chat_history()` - 调用LLM对聊天历史进行总结

2. **压缩触发机制**
   - 当非系统消息超过10条（5轮对话）时触发
   - 当聊天历史长度超过3000字符时触发

**技术实现：**
- 基于LLM的自动总结功能
- 智能分割聊天历史，保留最近的对话内容
- 流式输出总结过程，提升用户体验

### practice03/chat_client_v2.py

**功能用途：**
- 基于practice03的聊天界面
- 实现每五次聊天提取关键信息的功能
- 按照5W规则（Who、What、When、Where、Why）提取关键信息
- 将关键信息记录到本地电脑D:\chat-log\log.txt文件
- 开发新的function call，支持查找聊天历史功能

**核心功能模块：**

1. **关键信息提取**
   - `extract_key_information()` - 调用LLM从聊天历史中提取关键信息
   - `record_key_information()` - 将提取的关键信息记录到本地文件

2. **聊天历史查询**
   - `search_chat_history()` - 查找并返回聊天历史记录

3. **触发机制**
   - 每五次聊天自动提取一次关键信息
   - 当用户发送以"/search"开头的消息或表达查找聊天历史的意思时，触发聊天历史查询

**技术实现：**
- 基于LLM的信息提取功能
- 自动创建目录和文件，实现增量更新
- 新的function call工具，支持聊天历史查询
- 保持了原有的聊天历史压缩功能

### practice04/chat_client.py

**功能用途：**
- 基于practice03的聊天界面
- 集成AnythingLLM文档仓库查询功能
- 当用户提到"文档仓库"、"文件仓库"、"仓库"时，自动触发查询
- 使用API密钥进行认证，确保安全访问

**核心功能模块：**

1. **AnythingLLM查询**
   - `anythingllm_query()` - 调用AnythingLLM API查询文档仓库
   - 支持通过subprocess模块调用curl命令
   - 从.env文件读取API密钥和工作区标识

2. **触发机制**
   - 当用户提到"文档仓库"、"文件仓库"、"仓库"时，触发anythingllm_query工具

**技术实现：**
- 使用subprocess模块调用curl命令
- 访问http://localhost:3001/api/v1/workspace/{workspace_slug}/chat接口
- 使用API密钥进行认证
- 从.env文件读取配置变量
- 保持了原有的聊天历史压缩和关键信息提取功能

## 环境配置

1. **安装Python 3.12+**
2. **安装LMStudio**并下载一个LLM模型
3. **启动LMStudio本地服务器**（默认端口1234）
4. **复制配置文件**：
   ```bash
   copy env.example .env
   ```
5. **编辑.env文件**，填写模型名称
